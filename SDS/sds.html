Introduction

This document will define the design of the virtual classroom. It contains specific information about the expected input, output, classes, and functions. We have implemented Web and Android Application for Virtual Classroom. It has many functionality for both students and teachers. The purposes of this virtual classroom is To Keep the Students and Teachers Connected through a Virtual Platform. Through this app student can join their class with class code and teacher can send class code by email. We can also apply AI implementation and others advance functionality.

Description of Design Components

Student and Teacher: This is a key component. The main users are the Student and Teacher. Any outsider is not allowed. This contains the account selection which seperates who is who. Depending on this the profile will be created and they will be able access different functions.

Profile: A certain profile is set based on login information which contains each user's detail information (name, Email ,etc.). This profile is visible from the client side. User can also change their information anytime.

pdf handler: Use this component to update, view and edit pdf files. Client side can use this component to upload pdf on the server.

User Authentication: Checks User information is correct and allowes then to login or signup. Server side handles this operation. We use Django Auth to authenticate users.

AI Face Recognition: Artificial Intelligence checks if user is a Student or someone else. Server side handles this operation. We use Cmake and OpenCV to recognise user.

Voice Recorder: This is used to record voice from the system. Students are allowed to use this operation to record their voice which will be saved into the database.

Android App: An application in android phone connected to the server. Supposed to do everything that a user can to in website.

REST API: Allows us to manage/create GET or POST request from Android.

SQL Database: MySQL database is used to store all the information in the server. This is also an important component. Our entire website depends on MYSQL database or their will be errors.

    ER Diagram







    Table description and purposes


    Image_verify: Image_verify stores images of students, it has a relationship of one to many with Student table.

    User: Stores necessary info about the users so we can identify which user has which rules easily. A user can either be a student or a teacher.

    Student: The Student table is required to store information about the students. Student is a user so it has a one to one relationship with User table.

    Teacher: Stores Teacher's information. This table contains detailed information about the teacher.

    Classroom: Classroom has all the classroom information we need. A teacher can take many class. A class has many to many relationship with Student table. Because there can be many students in one class and one student in many class.

    Class_Metarials: All the provided pdf files and classroom metarials will be store here so we van easily access them. A class can have many metarials assigned.

    Questions: Students can ask many question about any class metarials. So we will store their questions in this table. Questions table has a one to many relationship with Questions table since there could be many questions about a certain class metarial
